# TEGA-D-001 — Черновик сообщения появляется в чате Telegram после его удаления в десктопной и мобильных версиях клиента.

## Summary
Удалённый черновик сообщения в Telegram периодически появляется снова спустя ~24 часа, несмотря на многократное удаление на мобильной и десктопной версии.

## Project
Telegram

## Component
Draft synchronization

## Version
- Desktop: **6.3.6.0**
- Android: **v12.1.1 (6211) direct arm64-v8a**
- OS: **Android 12**

## Severity
S3 — Major

## Priority
P2 — Medium

## Environment
- Telegram Desktop 6.3.6.0  
- Telegram Android v12.1.1 (6211), Android 12  
- Стабильное интернет-соединение  
- Telegram Web не используется  
- Баг проявляется в одном из чатов, тип чата на воспроизведение не влияет  

## Steps to Reproduce
1. Создать черновик сообщения в любом чате Telegram.
2. Удалить текст черновика в мобильном приложении Android.
3. Открыть Telegram Desktop и убедиться, что черновик отсутствует.
4. Закрыть приложения, продолжить обычное использование Telegram.
5. Спустя ~24 часа открыть тот же чат.
6. Обратить внимание: ранее удалённый черновик снова появился в поле ввода.
7. Удалить черновик повторно.
8. Через сутки черновик снова появляется. Поведение повторяемое.

## Expected Result
- Удалённый черновик должен исчезать окончательно.
- Все клиенты Telegram должны корректно синхронизировать факт удаления.
- Содержание черновика не должно появляться без действий пользователя.

## Actual Result
- Удалённый текст самопроизвольно возвращается в поле ввода чата спустя ~сутки.
- Проявляется и на Android, и на Desktop.
- Вероятная проблема в облачной синхронизации черновиков.
